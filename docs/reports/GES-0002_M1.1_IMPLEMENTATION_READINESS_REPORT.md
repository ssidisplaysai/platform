## 5. Architectural Conflicts

1. Relationship taxonomy discrepancy between BGS-0001 and BGC-0001.
- BGS-0001 includes Inheritance.
- BGC-0001 includes Association, Responsibility, Constraint.
- Impact: initial Business Genome relationship contract must use the narrower intersection set until governance clarifies.
- Required action: architecture clarification through governance process before expanding relationship taxonomy.

2. Compiler Core input/output naming currently reflects Discovery + Evidence orchestration.
- Current CompilerCore input contract receives KnowledgeSource and returns discovery artifacts plus Evidence IR.
- This is not an architecture violation for GES-0001, but M1.2 should introduce genome-stage input/output contracts as explicit pass-boundary artifacts.

## 9. Risk Register

1. Architecture drift
- Risk: introducing business semantics outside frozen BGC/BGS/GCC boundaries
- Mitigation: contract-only scope and explicit boundary tests

2. Hidden inference
- Risk: accidental inference logic in contracts or helpers
- Mitigation: no inference functions in M1.1 namespace

3. Provenance loss
- Risk: dropping evidence references in semantic contracts
- Mitigation: mandatory provenance contract fields and test assertions

4. Nondeterministic ordering
- Risk: unordered object iteration causing divergent outputs
- Mitigation: deterministic graph ordering helpers and deterministic serialization tests

5. Duplicate identities
- Risk: non-canonical identity assignment collisions
- Mitigation: deterministic identity input requirements and conflict context fields

6. Runtime contamination
- Risk: accidental imports from runtime or application layers
- Mitigation: dependency boundary tests and narrow exports

7. Premature abstraction
- Risk: introducing compile-time architecture patterns before evidence-backed need
- Mitigation: keep contracts minimal and immutable

8. Conflicting semantic evidence
- Risk: unresolved conflict handling policies across specifications
- Mitigation: preserve explicit conflict context; do not silently resolve
