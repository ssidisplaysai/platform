## 5. Merge Rules Implemented

### 5.1 Rule Catalog (2 Deterministic Rules)

| Rule ID | Matching Criteria | Description | Rationale Code |
|---------|-------------------|-------------|-----------------|
| `bgc.consolidation.rule.identical-class-and-designation` | Identical semantic class + identical normalized designation | Consolidates candidates with same meaning and name | BGC-RATIONALE-CONSOLIDATION-001 |
| `bgc.consolidation.rule.identical-semantic-identity` | Identical semantic class + identical semantic identity hash | Consolidates candidates with identical hash | BGC-RATIONALE-CONSOLIDATION-002 |

### 5.2 Consolidation Cases

**Case 1: Identical Class + Identical Normalized Designation**
```
IF semanticClass1 == semanticClass2
AND normalize(designation1) == normalize(designation2)
AND no conflicts in either candidate
THEN consolidate into one ConsolidatedSemantic
```

**Case 2: Identical Class + Identical Semantic Identity**
```
IF semanticClass1 == semanticClass2
AND deterministicIdentity(candidate1) == deterministicIdentity(candidate2)
AND no conflicts in either candidate
THEN consolidate into one ConsolidatedSemantic
```

**Case 3: Different Classes**
```
IF semanticClass1 != semanticClass2
THEN DO NOT consolidate
RATIONALE: Different meanings, separate consolidations
```

**Case 4: Conflicting Candidate**
```
IF candidate.conflictReferences.length > 0
THEN DO NOT consolidate with other candidates
RATIONALE: Conflicts are uncertain; preserve for downstream governed stages
```

## 6. Determinism Mechanisms

### 6.1 Deterministic Normalization

```typescript
function normalizeDesignation(designation: string): string {
  return designation
    .toLowerCase()              // Case normalization
    .trim()                     // Leading/trailing whitespace
    .replace(/\s+/g, " ");      // Collapse internal spaces
}
```

**Example:** "Performance  SLA", "PERFORMANCE SLA", "performance sla" ã all normalize to "performance sla"

### 6.2 Deterministic Merge Grouping

Uses **union-find** algorithm with path compression:
1. Initialize: Each candidate is its own root
2. For each candidate pair: Check if merge rule applies
3. If merge rule applies: Union(root1, root2)
4. Result: Disjoint sets representing consolidation groups

**Guarantees:**
- Same candidates always produce same groups
- Order-independent (permutation invariant)
- Lexicographically sorted output
