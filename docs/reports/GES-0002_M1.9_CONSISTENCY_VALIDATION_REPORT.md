## Validation Rules (12 Invariants)

### Implemented Invariants

| # | Invariant | Code | Severity | Description |
|---|-----------|------|----------|-------------|
| 1 | Unique Node Identities | BGC-VALID-001 | Error | No duplicate node IDs |
| 2 | Unique Edge Identities | BGC-VALID-002 | Error | No duplicate edge IDs |
| 3 | Source Node Exists | BGC-VALID-003 | Error | Every edge has valid source |
| 4 | Target Node Exists | BGC-VALID-004 | Error | Every edge has valid target |
| 5 | Node Provenance Present | BGC-VALID-005 | Error | Every node has provenance |
| 6 | Edge Provenance Present | BGC-VALID-006 | Error | Every edge has provenance |
| 7 | Lineage Preserved | BGC-VALID-007 | Warning | Evidence lineage intact |
| 8 | Deterministic Ordering | BGC-VALID-008 | Error | Nodes/edges lexicographically sorted |
| 9 | Graph Identity Reproducible | BGC-VALID-009 | Error | Graph ID matches current content |
| 10 | Compiler Metadata Present | BGC-VALID-010 | Error | All required metadata fields |

### Validation Algorithm

```
For each invariant:
  1. Check constraint (no modification)
  2. If violated:
     a. Record violation with details
     b. Create diagnostic
     c. Continue to next invariant
  3. If no violation:
     a. Increment passed count
     b. Continue

After all invariants:
  1. Sort violations and diagnostics deterministically
  2. Compute validation summary
  3. Compute validation history checksum
  4. Return BusinessGenomeValidationResult with original graph
```

## Diagnostic Codes

### BGC-VALID Diagnostic Namespace

| Code | Severity | Message | Trigger |
|------|----------|---------|---------|
| BGC-VALID-001 | Error | Duplicate node identity | Multiple nodes with same ID |
| BGC-VALID-002 | Error | Duplicate edge identity | Multiple edges with same ID |
| BGC-VALID-003 | Error | Orphan source node | Edge source not in graph |
| BGC-VALID-004 | Error | Orphan target node | Edge target not in graph |
| BGC-VALID-005 | Error | Node missing provenance | Node has empty provenance |
| BGC-VALID-006 | Error | Edge missing provenance | Edge has empty provenance |
| BGC-VALID-007 | Warning | Lineage not preserved | Node/edge missing evidence lineage |
| BGC-VALID-008 | Error | Deterministic ordering violation | Nodes or edges not sorted |
| BGC-VALID-009 | Error | Graph identity not reproducible | Graph ID doesn't match content |
| BGC-VALID-010 | Error | Compiler metadata missing | Required fields empty |

### Diagnostic Ordering

All diagnostics are sorted deterministically by:
1. Code (lexicographic)
2. Pass ID (if present)
3. Artifact ID (if present)
4. Message (lexicographic)

This ensures identical validation runs produce byte-identical diagnostic output.

#### Key Methods

1. **execute(input, context)**
   - Orchestrates all 10 validation invariant checks
   - Returns non-fatal result with violations
   - Never modifies input graph
   - Sorts violations and diagnostics deterministically

2. **findDuplicates(items)**
   - Helper: identifies duplicate strings in array
   - Returns sorted unique duplicates
   - Used for node/edge identity validation

3. **arraysEqual(a, b)**
   - Helper: compares arrays element-by-element
   - Used for ordering validation checks

4. **buildValidationSummary(violations)**
   - Computes aggregate statistics from violations
   - Counts errors, warnings, info violations
   - Determines overall validation status

5. **deriveGraphIdentity(data)**
   - Computes deterministic graph ID from content
   - Uses SHA256 of sorted node/edge IDs
   - Enables reproducibility verification

6. **deriveValidationChecksum(data)**
   - Computes checksum of complete validation result
   - Enables validation history verification

7. **buildFatalResult(input, diagnostics)**
   - Handles initialization failures
   - Returns empty validation result with error diagnostics
