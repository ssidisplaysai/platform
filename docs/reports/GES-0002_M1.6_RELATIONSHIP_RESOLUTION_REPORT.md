### 5.1 Rule Catalog (4 Deterministic Rules)

| Rule ID | Matching Criteria | Description | Rationale Code |
|---------|-------------------|-------------|-----------------|
| `bgc.relationship.rule.customer-purchased-product` | Customer → Product + purchase evidence | Explicit sales/purchase transactions | BGC-RATIONALE-RELATIONSHIP-001 |
| `bgc.relationship.rule.policy-governs-process` | Policy → Process + governance evidence | Explicit policy requirements/controls | BGC-RATIONALE-RELATIONSHIP-002 |
| `bgc.relationship.rule.organization-owns-asset` | Organization → Asset + ownership evidence | Explicit ownership/allocation records | BGC-RATIONALE-RELATIONSHIP-003 |
| `bgc.relationship.rule.supplier-provides-product` | Supplier → Product + supply evidence | Explicit supply agreements/catalogs | BGC-RATIONALE-RELATIONSHIP-004 |

### 5.2 Relationship Cases

**Case 1: Supported Rule Match + Explicit Evidence**
```
IF rule exists for (sourceClass, targetClass)
AND sourceSemanticClass matches rule source
AND targetSemanticClass matches rule target
AND no conflicts in either semantic
AND explicit evidence signal detected
THEN create relationship
```

**Case 2: No Applicable Rule**
```
IF no rule defined for (sourceClass, targetClass)
THEN emit BGC-REL-003 diagnostic
EMIT no relationship
```

**Case 3: Missing Evidence**
```
IF rule exists but evidence signals absent
THEN emit BGC-REL-002 diagnostic
EMIT no relationship
RATIONALE: Only explicit evidence triggers relationships
```

**Case 4: Conflicting Semantic**
```
IF sourceSemanticClass.hasConflicts OR targetSemanticClass.hasConflicts
THEN emit BGC-REL-004 diagnostic
EMIT no relationship
RATIONALE: Conflicts require governed resolution
```

## 6. Determinism Mechanisms

### 6.1 Deterministic Evidence Signal Detection

```typescript
function hasEvidenceSignalForRelationship(
  source: ConsolidatedSemantic,
  target: ConsolidatedSemantic,
  rule: RelationshipRule,
): boolean {
  // rule.requiredEvidenceSignal: "purchase-transaction|order-fulfillment|sales-record"
  const signals = rule.requiredEvidenceSignal.split("|");
  
  // Check if any evidence item contains matching signal keywords
  for (const item of [...source.evidenceItemIds, ...target.evidenceItemIds]) {
    for (const signal of signals) {
      if (item.toLowerCase().includes(signal.toLowerCase())) {
        return true;  // Evidence signal found
      }
    }
  }
  
  return false;  // No evidence signal
}
```

**Guarantees:**
- Same evidence items always trigger same relationships
- No probabilistic matching
- Evidence signal detection deterministic

### 6.2 Deterministic Merge Grouping

**All pairs of consolidated semantics checked:**
- For each (source, target) pair where source ≠ target
- Check if applicable rule exists
- Check for conflicts
- Check for evidence
- If all pass: create relationship

**Order-independence:**
- Pair processing order irrelevant
- All relationships produced regardless of input order

### 6.3 Deterministic Identity Derivation
