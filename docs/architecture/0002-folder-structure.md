# Genesis Architecture Folder Structure

Status: Active reference  
Owner: Engineering Leadership

## Canonical Top-Level Structure

1. docs/architecture: architecture records and review artifacts.
2. docs/governance: process and repository governance controls.
3. docs/reports: program and sprint reports.
4. tools/genesis: compiler and runtime platform tooling.
5. src: application and platform code organization.
6. definitions/entity: business definition inputs.
7. genesis: standards, language, roadmap, and templates.

## Source Of Truth Rule

Repository root is authoritative unless explicitly superseded by a board-approved record.
