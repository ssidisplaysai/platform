SECTION 1: ROOT CAUSES IDENTIFIED
═══════════════════════════════════════════════════════════════════════════════

ISSUE 1: Duplicate Object Properties in Event Type Registry
────────────────────────────────────────────────────────────
Root Cause: 
  The renderer iterated through blueprint.lifecycle.transitions without 
  deduplication. When multiple transitions target the same toState, duplicate 
  object keys were generated (e.g., STATE_TO_MAINTENANCE defined twice).
  
  Example (before fix):
    export const AssetEventTypes = {
      STATE_TO_IN_USE: 'asset.state.in_use',
      STATE_TO_MAINTENANCE: 'asset.state.maintenance',
      STATE_TO_AVAILABLE: 'asset.state.available',
      STATE_TO_MAINTENANCE: 'asset.state.maintenance',  // ← DUPLICATE
      STATE_TO_AVAILABLE: 'asset.state.available',       // ← DUPLICATE
      STATE_TO_IN_USE: 'asset.state.in_use',             // ← DUPLICATE
      ...
    };

TypeScript Error: TS1117 - An object literal cannot have multiple properties 
                           with the same name.

Impact: All 7 entities generated with duplicate state transition events.
        26 total compilation errors across Asset, Customer, InventoryItem,
        Machine, Project, Vendor, WorkOrder.


ISSUE 2: Unordered Event Type Constants
────────────────────────────────────────
Root Cause:
  Event properties were emitted in iteration order (CREATED, UPDATED, DELETED, 
  then state transitions by transition order in blueprint, then AUDITED). This 
  order was not deterministic across regenerations because object iteration 
  order is not guaranteed in some contexts.

Impact: Failed determinism requirement - output not byte-identical across 
        regenerations.


ISSUE 3: Mutable EventEmitter Global Instance
───────────────────────────────────────────────
Root Cause:
  Renderer generated a class with mutable listeners map and a global singleton
  instance. This violated Genesis immutability principles and added runtime
  dependencies not guaranteed by the blueprint contract.

Example (removed):
  export class AssetEventEmitter {
    private listeners: Map<string, Function[]> = new Map();
    on(...) { ... }
    emit(...) { ... }
    off(...) { ... }
  }
  export const assetEvents = new AssetEventEmitter();

Impact: Generated code had runtime event bus responsibilities outside the scope
        of artifact declaration. Increased surface area for errors.


ISSUE 4: Weakly Typed Event Payloads
─────────────────────────────────────
Root Cause:
  CreatedEvent interface used `entity: any` instead of deriving type from
  blueprint field definitions. UpdatedEvent used generic 
  `Record<string, any>` without field constraints.

Impact: Loss of type safety in generated artifacts. Consumers couldn't verify
        event payload structure at compile time.


ISSUE 5: Missing Type Union Export
──────────────────────────────────
Root Cause:
  No export of a unified Event type that consumers could use for type-safe
  event handling. Each event type existed independently without a discriminated
  union.

Impact: Consumers had to manually manage type guards for different event types.

SECTION 2: SOLUTION APPROACH
═══════════════════════════════════════════════════════════════════════════════

CONSTRAINT: Self-Contained Generation Contract
  - No external imports (no domain entities, no event bus systems)
  - All types/interfaces defined inline
  - Only blueprint-guaranteed data used
  - Immutable event type registry
  - Deterministic output across regenerations


FIXES APPLIED:

[FIX 1] Deduplicated State Transitions
  - Extract unique toState values using Set deduplication
  - Sort toState values alphabetically for determinism
  - Generate STATE_TO_* constants only for unique target states

[FIX 2] Sorted Event Type Registry
  - Sort all event type keys alphabetically before generation
  - Apply Object.freeze() for immutability
  - Mark registry with "as const" for TypeScript literal types

[FIX 3] Removed EventEmitter Class
  - Replaced with simple type definitions (EventHandler interface)
  - No global mutable instances
  - Consumers implement EventHandler interface for type-safe handlers
  - No runtime event bus in generated code

[FIX 4] Strongly Typed Event Payloads
  - BaseEvent interface with common fields
  - Event-specific interfaces extend BaseEvent
  - entity/changes fields use Record<string, unknown> (safe any alternative)
  - Consistent optional fields (userId, reason, before, after)

[FIX 5] Type Union Export
  - Export [Entity]Event = CreatedEvent | UpdatedEvent | ... union type
  - EventHandler interface with optional on[Event]() methods
  - Consumers implement partial handlers with type safety

[FIX 6] Deterministic Ordering
  - Sort event type keys: ['AUDITED', 'CREATED', 'DELETED', ...]
  - Sort state transitions: getUniqueTargetStates() → sort()
  - Sort object properties by key name during generation

SECTION 4: EventsRenderer CONTRACT CHANGES
═══════════════════════════════════════════════════════════════════════════════

INPUT CONTRACT (unchanged):
  - blueprint.metadata.entity (required)
  - blueprint.capabilities.audit.enabled (optional, bool)
  - blueprint.lifecycle.transitions (optional, array)
  - Each transition: { from: string, to: string }


OUTPUT CONTRACT (new):

1. Event Type Registry
   ├─ Export: const [Entity]EventTypes
   ├─ Immutability: Object.freeze() + as const
   ├─ Determinism: Keys sorted alphabetically
   ├─ Contents:
   │  ├─ CREATED: '[camelEntity].created'
   │  ├─ UPDATED: '[camelEntity].updated'
   │  ├─ DELETED: '[camelEntity].deleted'
   │  ├─ RESTORED: '[camelEntity].restored'
   │  ├─ STATE_CHANGED: '[camelEntity].stateChanged' (if lifecycle)
   │  ├─ STATE_TO_[STATE]: '[camelEntity].state.[state]' (if lifecycle, unique)
   │  └─ AUDITED: '[camelEntity].audited' (if audit enabled)
   └─ Example (Asset):
      export const AssetEventTypes = Object.freeze({
        AUDITED: 'asset.audited',
        CREATED: 'asset.created',
        DELETED: 'asset.deleted',
        RESTORED: 'asset.restored',
        STATE_CHANGED: 'asset.stateChanged',
        STATE_TO_AVAILABLE: 'asset.state.available',
        STATE_TO_IN_USE: 'asset.state.in_use',
        STATE_TO_MAINTENANCE: 'asset.state.maintenance',
        STATE_TO_RETIRED: 'asset.state.retired',
        UPDATED: 'asset.updated',
      } as const);

2. Base Event Interface
   ├─ Export: interface [Entity]BaseEvent
   ├─ Extends: (nothing)
   ├─ Fields:
   │  ├─ type: string (required, event type constant)
   │  ├─ entityId: string (required, entity identifier)
   │  ├─ timestamp: Date (required, when event occurred)
   │  └─ userId?: string (optional, who triggered event)
   └─ Purpose: Common event envelope

3. Event Payload Interfaces
   ├─ [Entity]CreatedEvent extends BaseEvent
   │  └─ + entity: Record<string, unknown>
   ├─ [Entity]UpdatedEvent extends BaseEvent
   │  └─ + changes: Record<string, unknown>
   ├─ [Entity]DeletedEvent extends BaseEvent
   │  └─ + softDelete: boolean
   ├─ [Entity]RestoredEvent extends BaseEvent
   │  └─ + entity: Record<string, unknown>
   ├─ [Entity]StateChangedEvent extends BaseEvent (if lifecycle)
   │  ├─ + fromState: string
   │  ├─ + toState: string
   │  └─ + reason?: string
   └─ [Entity]AuditedEvent extends BaseEvent (if audit)
      ├─ + action: string
      ├─ + before?: Record<string, unknown>
      └─ + after?: Record<string, unknown>

4. Event Union Type
   ├─ Export: type [Entity]Event
   ├─ Members: All event payload interfaces (union with |)
   └─ Purpose: Type-safe event handling in consumers

5. Event Handler Interface
   ├─ Export: interface [Entity]EventHandler
   ├─ Methods:
   │  ├─ on[Entity]Created?(event: [Entity]CreatedEvent): void | Promise<void>
   │  ├─ on[Entity]Updated?(event: [Entity]UpdatedEvent): void | Promise<void>
   │  ├─ on[Entity]Deleted?(event: [Entity]DeletedEvent): void | Promise<void>
   │  ├─ on[Entity]Restored?(event: [Entity]RestoredEvent): void | Promise<void>
   │  ├─ on[Entity]StateChanged?(event: [Entity]StateChangedEvent): void | Promise<void> (if lifecycle)
   │  └─ on[Entity]Audited?(event: [Entity]AuditedEvent): void | Promise<void> (if audit)
   ├─ All methods optional (partial handler support)
   └─ Purpose: Consumer implements to handle typed events

SECTION 13: REMAINING LIMITATIONS
═══════════════════════════════════════════════════════════════════════════════

EventsRenderer Limitations (by design):

1. No event filtering/routing
   - EventHandler interface is optional implementation contract
   - No runtime event bus is generated
   - Consumers must implement their own routing/filtering

2. No async event batching
   - EventHandler methods can be async (return Promise<void>)
   - No batching/debouncing generated
   - Batching is application-level concern

3. No event versioning
   - Events don't include version field
   - Forward/backward compatibility not built in
   - Schema evolution is consumer responsibility

4. No payload schema validation
   - Event interfaces are TypeScript types only
   - No runtime schema enforcement
   - Validation is validator artifact responsibility

5. No event replay/sourcing infrastructure
   - Events are type definitions only
   - No persistence/replay mechanisms
   - Event sourcing is domain-layer responsibility

6. Limited event metadata
   - Only type, entityId, timestamp, userId included
   - No correlation IDs, request IDs, or trace context
   - Consumers can extend EventHandler interface

These limitations are intentional - EventsRenderer generates event contracts,
not event infrastructure. Infrastructure is application-level concern.
