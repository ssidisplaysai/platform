# Genesis Canonical Library

## Purpose of /genesis

The /genesis directory is the canonical knowledge layer of this platform.
It contains foundational doctrine, discovery records, and governance method that define what can become canonical in architecture and implementation.

This directory is not a general note space and not a backlog of ideas.
It is the source of truth for foundational principles and their promotion lifecycle.

## /genesis vs /docs

/genesis:
- Holds doctrine, first principles, discovery logs, and canonical governance method.
- Defines what is true enough to constrain architecture, compiler behavior, and generation.
- Changes only through evidence-based promotion discipline.

/docs:
- Holds operational architecture records, implementation guidance, runbooks, and delivery artifacts.
- Describes decisions and execution state.
- Can evolve faster than /genesis.

In short: /genesis defines canonical truth criteria, while /docs records decisions and execution around those criteria.

## Governance Expectations

- Every meaningful addition must be grounded in explicit observation and evidence.
- Claims without traceable evidence cannot be promoted to canonical principle.
- Discovery, review, and validation are required before principles constrain implementation.
- Contradictions must be documented and resolved through the governance process, not by opinion.

## Change Discipline

- Prefer small, auditable changes.
- Record rationale, evidence basis, and promotion intent for each material update.
- Preserve history of discoveries and principle evolution.
- Do not rewrite foundational files as marketing narratives.
- Canonical promotion requires review and validation gates defined in governance.

## Relationship to Platform Components

- Compiler: /genesis defines the principles and promotion rules the compiler must obey.
- Runtime: /genesis constrains runtime behavior where truth, lineage, and verification rules apply.
- SDK: /genesis informs stable developer-facing contracts derived from canonical principles.
- AI agents: /genesis defines evidence and governance boundaries for autonomous reasoning and actions.
- Generated systems: /genesis is upstream of generation quality; generated outputs are projections of canonical knowledge, not sources of canonical truth.
