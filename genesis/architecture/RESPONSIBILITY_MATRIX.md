# Responsibility Matrix

Status: Approved
Classification: Genesis Standard
Type: Layer Responsibility Matrix

| Layer | Purpose | Question Answered | Primary Outputs | Primary Consumers |
|---|---|---|---|---|
| Reality | Provide the source domain independent of software | What exists? | Observations | Research |
| Research | Test hypotheses with evidence and method discipline | What might be true? | Findings, experiments, confidence notes | Standards, Architecture Governance |
| Standards | Stabilize canonical definitions and constraints | What is approved and stable? | Specifications, canonical rules | Compiler Specifications, Genome Specifications, Blueprint and Runtime Specifications |
| Compiler | Transform governed inputs deterministically | How is validated understanding transformed? | Auditable intermediate representations, validation results | Enterprise Genome |
| Enterprise Genome | Maintain canonical enterprise structural state and history | What is the validated enterprise model? | Genome state, identity and lineage structures | Blueprint |
| Blueprint | Project genome state into implementation-oriented forms | What should be built and configured? | Blueprint definitions and projection artifacts | Applications, Runtime Preparation |
| Runtime | Execute projected systems in operational context | What is happening now in operation? | Execution traces, events, outcomes | Living Enterprise loop, Observation pipelines |
| Living Enterprise | Close learning loop from execution back to research | What changed and what must be relearned? | New observations, improvement triggers | Research, Standards Evolution |
