# Git Workflow

## Principles

- Work from short-lived feature branches.
- Keep commits focused and descriptive.
- Merge only after review and verification.
- Preserve documentation alongside implementation changes.

## Workflow

1. Create a branch for the work.
2. Make focused changes.
3. Verify relevant tests and checks.
4. Document architecture or process changes.
5. Merge only when the work is complete and reviewable.
