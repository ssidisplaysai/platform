# Genesis OS Engineering Handbook

This handbook captures the engineering structure for Genesis OS as a metadata-driven enterprise operating system. It is intended to guide architecture, delivery, operations, and module development across the platform.

## Scope

- Preserve Clean Architecture boundaries.
- Keep business logic in services.
- Keep data access in repositories.
- Treat the runtime as the orchestration layer.
- Keep UI as presentation only.
- Use metadata as the primary mechanism for enterprise behavior.

## Handbook Structure

- Vision and principles
- Roadmap and delivery planning
- Architecture and standards
- Development practices
- Operating system behavior
- Sprint planning and templates

## Operating Principles

1. Enterprise-first design.
2. No avoidable technical debt.
3. Domain logic remains in the domain and service layers.
4. Runtime behavior is explicit and orchestrated.
5. Every decision should support long-term platform health.
