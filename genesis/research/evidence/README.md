# Research Evidence Policy

Status: Approved
Classification: Genesis Research Process

## Purpose

Define how research evidence is collected, qualified, and evaluated before findings can influence standards or implementation specifications.

## Acceptable Evidence Types

- source-attributed operational records
- audited event logs with provenance
- reproducible experiment outputs
- versioned models with lineage metadata
- controlled comparative studies with traceable datasets

## Source Requirements

All evidence must include:
- source identity
- collection timestamp or version marker
- context of collection
- integrity notes where applicable

## Lineage Requirements

Evidence must preserve:
- transformation path from raw source to analyzed artifact
- intermediate processing steps
- responsible agent or process identity per step

## Provenance Requirements

Evidence packages must show:
- origin system or collection method
- ownership or stewardship context
- immutability or change log details

## Confidence Notes

Each evidence item should include:
- confidence level
- uncertainty factors
- known limitations and possible bias

## Invalid or Weak Evidence Examples

- unattributed screenshots or excerpts
- claims without source references
- transformed data without lineage
- synthetic examples presented as factual observations
- stale evidence with unknown collection context

## Evidence Quality Gate

Research findings cannot advance when evidence quality is below minimum source, lineage, and provenance requirements.
