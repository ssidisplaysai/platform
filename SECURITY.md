# Genesis Security Policy

## Scope

This policy applies to all source code, architecture records, tooling, and documentation in this repository.

## Reporting A Security Issue

1. Do not open public issues for exploitable security vulnerabilities.
2. Report security concerns to the security owner listed in [ENGINEERING_CONTACTS.md](ENGINEERING_CONTACTS.md).
3. Include reproduction details, impact assessment, and affected paths.

## Response Targets

1. Initial acknowledgement: within 1 business day.
2. Triage and severity classification: within 3 business days.
3. Mitigation plan: within 5 business days for high/critical items.

## Security Engineering Principles

1. Preserve tenant and permission boundaries.
2. Protect compilation and promotion trust boundaries.
3. Keep audit and provenance integrity intact.
4. Treat plugin and extension points as high-risk interfaces.

## Disclosure Policy

1. Coordinate disclosure timing with security and architecture owners.
2. Do not disclose exploit details before validated mitigation is available.
